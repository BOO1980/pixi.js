Loading images into the texture cache

  • Because Pixi renders the image on the GPU with WebGL, the image needs to be in a format that the GPU can process
  • Before you can make a sprite display an image, you need to convert an ordinary image file into a WebGL texture
  • To keep everything working fast and efficiently, Pixi uses a texture cache to store and reference all the textures your sprites will need.
  • The name of the textures are strings that match the file locations of the images they refer to
  • The textures are stored in a WebGL compatible format thats efficient for Pixi renderer to work with.
  • You can use the Pixis Sprite class to make a new sprite using the texture
  • To load the image file and convert into a texture, Pixi's built in loader object is used and a function called setup is called

Monitoring load progress

  • Pixi's loader has a special progress event that will call a customizable function that will run each time a file loads
  • Each time one of the files load, the progress event calls loadProgressHandler to display "loading" in the console.
  • You can also find out exactly which file has loaded and what percentage of overall files are have currently loaded by adding optional loader and resource parameters to the loadProgressHandler
  • You can use this as the basis for creating a loading progress bar
  • There are additional properties you access on the resource object. resource.error will tell you any possible error that happened while trying to load file.
  • resource.data lets you access the file's raw binary data
  • Pixi's loader is feature-rich and configurable
  • The loads chainable add method takes 4 basic arguments
    • name (string): The name of the resource to load. If it's not passed, the url is used
    • url (string): the url for this resource, relative to the baseUrl of the loader
    • optionObject:
      • options (object literal): The options for load
      • options.crossOrigin:(Boolean): is the request cross-origin? The default is to determine automatically
      • options.loadType: How should the resource be loaded? The default value is Resource.LOAD_TYPE.XHR
      • options.xhrType: How should the data being loaded be interpreted when using XHR? The default value is Resource.XHR_RESPONSE_TYPE.DEFAULT
